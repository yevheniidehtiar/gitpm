## Motivation
When Claude Code opens a repo, it reads CLAUDE.md for context. GitPM can auto-generate a project context section from .meta/ that tells Claude about milestones, active epics, current sprint, and key stories.

## Proposed Solution
`gitpm context` command that outputs a markdown section for CLAUDE.md:
```markdown
## Project Status (auto-generated by gitpm)
### Active Milestone: v0.7.0 — High-Volume Scalability (65% done)
### Current Sprint: 2026-W14 (8/15 stories done)
### Active Epics:
- Adapter Query Performance (12/16 stories done)
- Load Testing Suite (2/9 stories done)
### Blocked:
- keyset pagination (blocked by: cursor params in BaseAdapter)
```

Auto-update via git pre-commit hook or `gitpm sync --update-claude-md`.

## Impact
This is the bridge between 'files on disk' and 'AI agent understands the project'. Makes .meta/ actually useful for Claude Code workflows.
